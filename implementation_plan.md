# 영어 단어장 웹 애플리케이션 및 알림 시스템 구현 계획 (English Vocabulary App & Notification System Plan)

사용자가 요청한 초6~중3 수준의 영어 단어장 기능에 더해, **부모님(보호자) 모니터링 시스템**과 **SMS/메신저 자동 알림 기능**을 통합하는 계획입니다. 이 기능을 구현하기 위해서는 기존의 '순수 프론트엔드(브라우저) 기반' 구조를 넘어, 데이터베이스와 서버(백엔드)가 필수적입니다.

## User Review Required

> [!CAUTION]
> **⚠️ 중대한 아키텍처 변경 알림 (매우 중요)**
> - 기존에는 브라우저에서만 도는 아주 가벼운 버전을 기획했으나, **'매일 저녁 9시 알림'**과 **'부모님께 결과 메시지 전송'**, 그리고 부모님이 다른 기기에서 **'결과 페이지 확인'**을 하시려면 중앙화된 **서버(Backend)와 데이터베이스(DB)**가 반드시 필요합니다.
> - 문자 메시지(SMS) 전송을 위해서는 **Twilio, Solapi(쿨에스엠에스) 또는 알림톡 API**와 같이 유료 (혹은 무료 체험판) 외부 서비스 연동이 필요합니다.
>
> 💡 **제안하는 기술 스택 변경 (선택 필요):**
> 1. **Next.js + Firebase (또는 Supabase) + SMS API 통합**: 무료로 서버 호스팅(Vercel)과 DB를 구축할 수 있으며 단어장 기능과 메시지 전송 기능을 통합하기 좋습니다.
> 2. **Node.js(Express) + 순수 HTML/JS**: 사용자의 컴퓨터나 저렴한 가상 서버를 24시간 켜두고 서버로 구동하는 방식입니다.
>
> 현재 단계에서는 어떤 방식이 더 편하신가요? 잘 모르시겠다면 1번(Next.js 풀스택 구조)으로 기초 틀을 짜서 브라우저 기반으로 먼저 구동해 볼 수 있도록 진행하겠습니다.
